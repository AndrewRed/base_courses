Обзор интерфейса n8n

Интерфейс n8n интуитивно понятен и позволяет эффективно работать с workflow. Давайте изучим основные элементы.

Главная панель

После входа в n8n вы увидите главную панель, которая содержит:

  • Список workflow — все созданные вами автоматизации
  • Кнопка "New Workflow" — создание нового workflow
  • Поиск — быстрый поиск по workflow
  • Настройки — конфигурация n8n

Редактор workflow

При открытии или создании workflow вы попадаете в визуальный редактор, который состоит из:

1. Canvas (Холст)

Центральная область, где вы размещаете и соединяете ноды. Это рабочее пространство для создания вашей автоматизации.

2. Панель нод (Nodes Panel)

Слева находится панель с доступными нодами, организованными по категориям:

  • Core — базовые ноды (Start, Set, IF, Switch и др.)
  • Integrations — интеграции с внешними сервисами
  • AI — ноды для работы с AI и LLM
  • Code — выполнение JavaScript/Python кода

3. Панель свойств (Properties Panel)

Справа отображаются свойства выбранной ноды, где вы можете:

  • Настраивать параметры ноды
  • Просматривать входные и выходные данные
  • Тестировать ноду

Основные типы нод

Триггеры (Triggers)

Ноды, которые запускают workflow. Примеры:

  • Webhook — принимает HTTP запросы
  • Schedule — запуск по расписанию
  • Manual — ручной запуск

Действия (Actions)

Ноды, которые выполняют определенные действия:

  • HTTP Request — отправка HTTP запросов
  • Set — установка значений
  • Code — выполнение пользовательского кода

Логические ноды

Ноды для управления потоком:

  • IF — условное ветвление
  • Switch — множественное ветвление
  • Merge — объединение потоков

Работа с данными

В n8n данные передаются между нодами в виде JSON объектов. Вы можете:

  • Обращаться к данным через выражения: {{ $json.field }}
  • Использовать функции для трансформации: {{ $json.field.toUpperCase() }}
  • Работать с массивами и объектами

Тестирование workflow

n8n предоставляет мощные инструменты для тестирования:

  • Test workflow — запуск workflow в тестовом режиме
  • Execute Node — выполнение отдельной ноды
  • View execution data — просмотр данных на каждом этапе
