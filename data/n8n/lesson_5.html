Первоначальная настройка

После установки n8n необходимо выполнить первоначальную настройку для комфортной работы.

1. Создание учетной записи

При первом запуске n8n предложит создать учетную запись:

  • Введите email и пароль
  • Запомните эти данные — они понадобятся для входа
  • Для production используйте надежный пароль

2. Настройка окружения

Переменные окружения

Важные переменные для настройки:

# Базовые настройки
N8N_HOST=your-domain.com
N8N_PORT=5678
N8N_PROTOCOL=https

# Безопасность
N8N_BASIC_AUTH_ACTIVE=true
N8N_BASIC_AUTH_USER=admin
N8N_BASIC_AUTH_PASSWORD=secure_password

# База данных (опционально)
DB_TYPE=postgresdb
DB_POSTGRESDB_HOST=localhost
DB_POSTGRESDB_PORT=5432
DB_POSTGRESDB_DATABASE=n8n
DB_POSTGRESDB_USER=n8n
DB_POSTGRESDB_PASSWORD=password

# Email (для уведомлений)
N8N_EMAIL_MODE=smtp
N8N_SMTP_HOST=smtp.gmail.com
N8N_SMTP_PORT=587
N8N_SMTP_USER=your_email@gmail.com
N8N_SMTP_PASS=your_password

3. Настройка в интерфейсе

Settings (Настройки)

В меню Settings доступны следующие разделы:

  • Personal — личные настройки профиля
  • API — управление API ключами
  • Community Nodes — установка дополнительных нод
  • Security — настройки безопасности

Создание API ключа

API ключ необходим для:

  • Программного управления workflow
  • Интеграции с внешними системами
  • Автоматизации через API

Создание:

  1. Перейдите в Settings → API
  2. Нажмите "Create API Key"
  3. Сохраните ключ в безопасном месте

4. Установка дополнительных нод

n8n поддерживает community nodes — дополнительные ноды, созданные сообществом.

Установка через интерфейс:

  1. Settings → Community Nodes
  2. Введите имя пакета (например, @n8n/n8n-nodes-langchain)
  3. Нажмите "Install"

5. Настройка вебхуков

Для работы с webhooks необходимо:

  • Настроить правильный HOST и PROTOCOL
  • Убедиться, что порт доступен извне (для production)
  • Настроить SSL сертификат для HTTPS

6. Резервное копирование

Настройте автоматическое резервное копирование:

  • Данные хранятся в ~/.n8n (по умолчанию)
  • Регулярно создавайте копии этой папки
  • Используйте экспорт workflow через интерфейс

7. Оптимизация производительности

Для Docker

docker run -d \
  --name n8n \
  --memory="2g" \
  --cpus="2" \
  -p 5678:5678 \
  n8nio/n8n

Переменные окружения для оптимизации

N8N_METRICS=true
N8N_LOG_LEVEL=info
EXECUTIONS_PROCESS=main
EXECUTIONS_DATA_SAVE_ON_ERROR=all
EXECUTIONS_DATA_SAVE_ON_SUCCESS=all

Проверка настройки

Убедитесь, что:

  • Вы можете войти в систему
  • Создан API ключ
  • Настроены переменные окружения
  • Доступны необходимые ноды

В следующем уроке мы настроим рабочее окружение для разработки.
