Создание первого workflow

Давайте создадим ваш первый workflow в n8n. Мы сделаем простую автоматизацию, которая поможет понять основные принципы работы.

Шаг 1: Создание нового workflow

  1. На главной странице нажмите "New Workflow"
  2. Дайте workflow понятное имя, например "Мой первый workflow"
  3. Откроется редактор с пустым canvas

Шаг 2: Добавление триггера

Каждый workflow начинается с триггера — ноды, которая запускает выполнение.

  1. В панели нод слева найдите категорию "Core Nodes"
  2. Перетащите ноду Manual Trigger на canvas
  3. Эта нода позволяет запускать workflow вручную для тестирования

Шаг 3: Добавление действия

Теперь добавим ноду, которая что-то делает:

  1. Найдите ноду Set в категории "Core Nodes"
  2. Перетащите её на canvas
  3. Соедините выход Manual Trigger с входом Set (перетащите от одной ноды к другой)

Шаг 4: Настройка ноды Set

Нода Set позволяет устанавливать значения данных:

  1. Кликните на ноду Set
  2. В панели справа нажмите "Add Value"
  3. В поле "Name" введите: message
  4. В поле "Value" введите: Привет из n8n!
  5. Нажмите "Execute Node" для тестирования

Шаг 5: Добавление HTTP Request

Добавим ноду для отправки HTTP запроса:

  1. Найдите ноду HTTP Request в категории "Core Nodes"
  2. Добавьте её после ноды Set
  3. Соедините ноды
  4. Настройте HTTP Request:
    • Method: GET
    • URL: https://httpbin.org/get

Шаг 6: Тестирование workflow

Теперь протестируем весь workflow:

  1. Нажмите кнопку "Execute Workflow" в правом верхнем углу
  2. Просмотрите данные на каждом этапе, кликая на ноды
  3. Убедитесь, что данные передаются корректно

Структура данных в n8n

Данные в n8n передаются в формате JSON. Каждая нода получает массив объектов:

[
  {
    "json": {
      "field1": "value1",
      "field2": "value2"
    },
    "binary": {},
    "pairedItem": {}
  }
]

Работа с выражениями

В n8n можно использовать выражения для доступа к данным:

  • {{ $json.field }} — доступ к полю из текущего элемента
  • {{ $json["field name"] }} — доступ к полю с пробелами
  • {{ $("Set").item.json.message }} — доступ к данным из другой ноды

Пример: Использование данных из предыдущей ноды

Создайте еще одну ноду Set и используйте данные из первой:

  1. Добавьте новую ноду Set
  2. В значении используйте выражение: {{ $json.message }}
  3. Это скопирует значение из предыдущей ноды

Сохранение workflow

Не забудьте сохранить workflow:

  • Нажмите Ctrl+S (или Cmd+S на Mac)
  • Или используйте кнопку "Save" в правом верхнем углу

Полезные советы

  • Используйте понятные имена для нод (двойной клик на ноде)
  • Комментируйте сложные части workflow
  • Тестируйте каждую ноду отдельно перед соединением
  • Используйте "Execute Node" для отладки

Поздравляем! Вы создали свой первый workflow. В следующем уроке мы изучим основные ноды и их использование.
